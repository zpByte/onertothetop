# ONER宇宙生存模拟器：模型返回协议 MVP

## 1. 目标

本协议用于把游戏从“写死事件库”升级为“可控的大模型 DM”。模型负责生成每日剧情、NPC 反应、行动反馈和捡手机片段；前端只接受严格 JSON，不直接渲染自由散文。

核心原则：

- 每一回合必须有新场景、新矛盾和明确选择。
- 玩家每次选择后必须得到后果反馈。
- 每个选择至少影响 2 项数值。
- 身份可以从初始身份流转成新的叙事位置。
- 模型不得声称现实艺人真实私生活、真实关系或真实争议。

## 2. 游戏状态输入

前端每回合向模型传入当前状态。

```json

  "game": "ONER宇宙生存模拟器",

  "max_turn": 30,
  "origin_identity": "品牌 PR",
  "current_role": "流程救火员",
  "stats": {

    "钱包": 61,
    "事业": 72,

```

## 6. 身份流转规则

初始身份只决定入口，不锁死路线。模型可以根据状态返回 `new_role`。

可用身份流转示例：

- `临时靠谱的人`
- `粉圈人情债务人`
- `流程救火员`
- `项目复盘临时工`
- `合作名单边缘人`
- `粉圈重点观察对象`
- `关系线高风险知情人`
- `隐形操盘手`
- `项目负责人苗子`
- `主动退场的人`

流转约束：

- `事业` 高且 `舆情` 中低时，身份应向工作/项目线移动。
- `关系` 高且 `隐蔽` 中高时，身份应向关系知情人移动。
- `舆情` 高且 `隐蔽` 低时，身份应向风险人物移动。
- `心态` 长期低时，应出现自我保护或退场路线。

## 7. 生成约束

模型必须遵守：

- 不写现实艺人的真实婚恋、家庭、私生活、争议或未经确认信息。
- 不把粉丝整体写成恶毒群体。
- 不把经纪人、公司、工作人员写成无脑反派。
- 不让关系无代价推进。
- 不让玩家每次选择都赢。
- 不提前透露玩家第一视角不知道的信息。
- 不输出超出 JSON 之外的解释文字。

## 8. 前端校验规则

前端收到模型返回后应校验：

- `type` 必须是允许值。
- `choices` 数量必须为 2-4。
- 每个 `choice.effects` 至少包含 2 个数值。
- 数值变化建议范围为 -15 到 +15。
- `scene.body` 长度建议为 80-220 字。
- `result_body` 长度建议为 60-180 字。
- 如果 JSON 校验失败，使用本地事件库兜底。
